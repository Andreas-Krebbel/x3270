Outstanding Bugs

  • c3270 does not work correctly in UTF-8 locales. In particular, characters in the range 0x80..0xff are displayed incorrectly (note that these refer to display character codes, not host EBCDIC codes).

    A simple workaround is to ensure that the terminal emulator running c3270 (xterm, gnome-terminal, etc.) is not using a UTF-8 locale. For example, you can set the environment variable LANG or LC_CTYPE to en_US.ISO8859-1 before starting the terminal emulator.

    A fix for this is in progress, but will be too large to release as a patch.

  • 'Flipped' mode (for right-to-left languages) is disabled.
  • File transfer does not support DBCS.
